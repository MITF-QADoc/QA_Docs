MITF Online Payment Ecosystem

Platform Vision

The MITF Online Payment Ecosystem replaces scattered bank-credential integrations with a unified layer: API keys, a merchant control plane, and a dual-auth payment API. External channels (WooCommerce, direct API, POS) connect through defined APIs while bank credentials stay in the portal vault. The core payment engine—OTP flows and bank adapters—remains unchanged.

Goal: Self-service onboarding, partner scale, and operational visibility without redesigning payments.

Core Ecosystem Components

Component Role
Online Payment API V2 Dual-auth host — JWT proxy to V1 + RESTful /v2/* for API-key clients
Legacy Core Thin gRPC bank gateway (Pay, GetBalance, CheckTransactionStatus)
Merchant Portal Control plane — web UI + API for identity, keys, operations, developer tools
WooCommerce & Payment Service E-commerce channel; API key only, hosted /pay page
Online Payment API V1 Frozen; preserved behind V2 JWT proxy

Key Capabilities

  • API keys — prefixed mitf_live_* / mitf_test_*; Argon2id at rest; portal vault for bank creds
  • Payment sessions — V2 owns OTP, settlement via Legacy Core, refunds, balance, transactions, reports
  • Control plane — idempotency, rate limits, quotas, audit; RFC 7807 errors on /v2/*
  • Merchant portal — bills, refunds, reporting, integrations, Developer Center (docs, tester, plugin)
  • Channels — WooCommerce plugin, direct REST API, POS via portal APIs

Business Value

Stakeholder Value
Merchants Fast onboarding, one portal, no creds in third-party systems
Partners REST API, sandbox keys, Swagger, in-portal tester
Operations Central key management, audit, quotas, unified reporting
Platform Scalable dual-auth host, V1 migration path, isolated bank gateway

Benefits for Merchants and Partners

Merchants: Sandbox keys, single portal for ops and integrations, bilingual UI, guided onboarding.

Partners: Stable /v2/* contracts, legacy JWT preserved via proxy, plugin from portal, documented trust boundaries.

System Architecture Summary

Channels (WooCommerce, Direct API, POS)
        │
        ├──▶ Merchant Portal (Web + API) — keys, vault, ops
        │
        └──▶ Online Payment V2 — JWT proxy → V1  |  /v2/* → Legacy Core → Bank

Principles: V2 executes new integrations; Legacy Core is the only bank gateway; portal issues and resolves keys; channels hold API keys, not bank secrets.

Summary

A production-ready platform that separates control plane from payment engine, vaults credentials centrally, and serves both legacy JWT and modern API-key clients—enabling merchants and partners to connect quickly without compromising security or payment reliability.
