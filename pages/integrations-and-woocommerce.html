Integrations & WooCommerce

Overview

The integration framework connects external systems—stores, custom apps, POS—to MITF via portal-issued API keys, not bank credentials. Keys authenticate against V2; credentials resolve server-side from the portal vault. OTP-based payment security is preserved across all channels.

Key Features

  • API keys — mitf_live_* / mitf_test_*; vault-backed credential resolution
  • V2 REST API — sessions, refunds, balance, transactions, reports
  • Control plane — idempotency, rate limits, quotas, audit
  • Developer tools — Swagger in portal, API tester, plugin download
  • Thin connectors — channels hold keys only; hosted /pay for sensitive customer input

Capabilities

Direct API: x-api-key auth on /v2/* — sessions, complete, refund, balance, transactions, reports. RFC 7807 errors.

Supporting services: Masarat Payment Service (WooCommerce checkout), Legacy Core (bank), Portal (resolve-key).

Plugin pattern: Portal-distributed, HMAC-signed initiation, no bank creds in third-party code.

WooCommerce Integration

Purpose: WordPress merchants accept MITF payments with an API key only—no bank creds in WordPress.

Features: Hosted /pay page (card ID + OTP), HMAC checkout, order status sync, sandbox/production keys, portal download (v0.4+).

Setup: Create integration → vault creds → issue key → download plugin → paste key in WooCommerce → test checkout.

Payment flow:

Checkout → Plugin (api_key + HMAC) → Payment Service → resolve-key (portal)
→ Hosted /pay (card ID + OTP) → V2 complete → Order paid

Trust boundary: plugin holds API key + HMAC secret only; bank creds never enter WordPress.

Business Value

  • Fast onboarding — sandbox keys, in-portal docs/tester, one-click plugin
  • Lower risk — no bank creds in third-party code; idempotency and quotas at the edge
  • Channel flexibility — direct API, WooCommerce, POS; same key + resolve pattern for future channels

Highlights

Strength Detail
Keys, not creds Portal vault; integrators authenticate with API keys
Thin WooCommerce plugin One key in WordPress; everything else server-side
Hosted payment page Card ID and OTP off merchant infrastructure

Summary

API keys replace bank secrets across the ecosystem. V2 provides the REST surface; WooCommerce and future connectors stay thin; the portal drives key lifecycle and developer self-service.
