Merchant Portal

Overview

The Merchant Portal is the control plane for the MITF Online Payment Ecosystem—a unified product combining a self-service web application (Angular) and backend API (.NET, FastEndpoints). Merchants onboard, manage payments, issue API keys, and connect integrations from one place. Bank credentials stay encrypted in the portal vault; integrators receive API keys only.

Key Features

  • Dashboard & onboarding — KPIs, guided setup, quick-access to key modules
  • Bills & payment links — create bills, share links, guest bill payment page
  • Refunds & event tracker — initiate refunds, audit platform events
  • Reporting — bills, POS, refunds, installments, settlement, Customer 360
  • Integrations & API keys — create integrations, vault MITF creds, issue/revoke keys (mitf_live_* / mitf_test_*)
  • Developer center — embedded API docs, interactive tester, WooCommerce plugin download
  • Users & POS — RBAC user management, POS terminal lifecycle and bulk import
  • Identity & security — JWT + OTP login, QR login (SignalR), permission-gated access
  • Notifications — in-app bell with unread counts

All modules are gated by role-based permissions; the UI is bilingual (Arabic/English).

Credential & Integration Control

The portal is the sole authority for API keys and bank credentials across the ecosystem:

  • Vault MITF credentials (AES-GCM encrypted) — never exposed to integrators
  • Issue sandbox/production keys (Argon2id hashed; plaintext shown once)
  • Revoke keys immediately with full audit trail
  • Internal resolve-key endpoint for V2 API and payment service (server-to-server)

Backend Services

Payment and POS operations proxy to Masarat services; the portal owns identity, keys, and reporting:

  • Bill creation, public payment links, installments, refunds
  • POS terminal CRUD, state, password reset, Excel bulk import
  • Customer profiles, notifications, event tracker, plugin release management

Business Value

  • Single product — operations, insights, and developer tools in one portal; no fragmented tooling
  • Self-service onboarding — sandbox keys, in-portal docs/tester, plugin download accelerate go-live
  • Security by design — bank PINs never leave the vault; integrators hold API keys only
  • Operational visibility — reports, event tracker, and key audit support compliance and support
  • Partner enablement — one credential authority for V2, payment service, WooCommerce, and POS

Summary

The Merchant Portal is the merchant-facing command center and credential authority for the ecosystem. Merchants operate payments, manage integrations, and onboard partners from one secure, self-service product—while bank secrets stay vaulted and the rest of the platform resolves access through API keys.
